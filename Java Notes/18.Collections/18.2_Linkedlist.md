# 18.2_LinkedList in Java

A **LinkedList** is a doubly linked list implementation of the `List` interface.

Unlike ArrayList:

```text
ArrayList  → Dynamic Array
LinkedList → Doubly Linked List
```

Visualization:

```text
[10] <-> [20] <-> [30] <-> [40]
```

Each node stores:

```text
Data
Previous Address
Next Address
```

---

# Creating LinkedList

```java
import java.util.LinkedList;

LinkedList<Integer> list =
    new LinkedList<>();
```

---

# ArrayList vs LinkedList

## Internal Structure

ArrayList:

```text
[10][20][30][40]
```

LinkedList:

```text
[10] <-> [20] <-> [30] <-> [40]
```

---

## Accessing Element

ArrayList:

```java
list.get(5000);
```

Direct access.

Fast.

```text
O(1)
```

---

LinkedList:

Must traverse.

```text
10 -> 20 -> 30 -> ...
```

Slow.

```text
O(n)
```

---

## Insertion in Middle

ArrayList:

```text
Shift elements
```

Example:

```text
[10,20,30,40]

Insert 25

[10,20,25,30,40]
```

Need shifting.

```text
O(n)
```

---

LinkedList:

Just change links.

```text
20 -> 30

becomes

20 -> 25 -> 30
```

```text
O(1)
```

(after reaching node)

---

## Deletion

ArrayList:

Shift elements.

```text
O(n)
```

LinkedList:

Change pointers.

```text
O(1)
```

(after reaching node)

---

# Complexity Table

| Operation     | ArrayList | LinkedList |
| ------------- | --------- | ---------- |
| get(i)        | O(1)      | O(n)       |
| set(i)        | O(1)      | O(n)       |
| add(end)      | O(1) avg  | O(1)       |
| add(start)    | O(n)      | O(1)       |
| remove(start) | O(n)      | O(1)       |
| search        | O(n)      | O(n)       |

---

# Memory Usage

ArrayList:

```text
Stores only data
```

Less memory.

---

LinkedList:

```text
Data
Previous Pointer
Next Pointer
```

More memory.

---

# When to Use ArrayList?

Use when:

```text
Frequent Reading
Random Access
get(i) used often
```

Examples:

* Student records
* Product lists
* LeetCode problems

Most Java developers use ArrayList by default.

---

# When to Use LinkedList?

Use when:

```text
Frequent Insertions
Frequent Deletions
Queue
Deque
Stack
```

Examples:

* Browser history
* Undo/Redo
* Music playlist navigation
* LRU Cache internals

---

# Interview Answer

### Which is faster?

Depends.

For:

```text
Reading → ArrayList
Insertion/Deletion → LinkedList
```

---

### Which is used more?

```text
ArrayList ≈ 90%+
LinkedList ≈ 10%
```

In DSA, LeetCode, Spring Boot, and most backend applications, **ArrayList is the first choice** unless you specifically need frequent insertions/deletions at the ends or queue/deque behavior.
