# 18.3_Vector in Java

## What is Vector?

Vector is a dynamic array similar to ArrayList.

```java
Vector<Integer> vec = new Vector<>();
```

Features:

* Dynamic size
* Ordered collection
* Allows duplicates
* Index-based access
* Thread-safe (Synchronized)

Example:

```java
Vector<Integer> vec = new Vector<>();

vec.add(10);
vec.add(20);
vec.add(30);

System.out.println(vec);
```

Output:

```text
[10, 20, 30]
```

---

# Why Vector?

Before Collections became popular, Java provided Vector as a dynamic array.

Today:

```text
ArrayList = Preferred
Vector = Legacy Class
```

# capacity()

Very important Vector method.

```java
vec.capacity();
```

Returns current capacity.

Example:

```java
Vector<Integer> vec =
new Vector<>();

System.out.println(
    vec.capacity()
);
```

Output:

```text
10
```

Default capacity = 10.

---

# ensureCapacity()

```java

```

---

# trimToSize()


# Iterating Vector

## For Loop

```java

    System.out.println(
        vec.get(i)
    );
}
```

## Enhanced For Loop

```java
for(int x : vec){
    System.out.println(x);
}
```

## Iterator

```java
Iterator<Integer> it =
vec.iterator();

while(it.hasNext()){
    System.out.println(
        it.next()
    );
```

# ArrayList vs LinkedList vs Vector

| Feature            | ArrayList     | LinkedList         | Vector        |
| ------------------ | ------------- | ------------------ | ------------- |
| Internal Structure | Dynamic Array | Doubly Linked List | Dynamic Array |
| Ordered            | Yes           | Yes                | Yes           |
| Duplicates         | Yes           | Yes                | Yes           |
| Thread Safe        | No            | No                 | Yes           |
| get(i)             | O(1)          | O(n)               | O(1)          |
| set(i)             | O(1)          | O(n)               | O(1)          |
| add(end)           | O(1) Avg      | O(1)               | O(1) Avg      |
| add(start)         | O(n)          | O(1)               | O(n)          |
| remove(start)      | O(n)          | O(1)               | O(n)          |
| Memory Usage       | Low           | High               | Low           |
| Performance        | Fast          | Moderate           | Slower        |
| Preferred Today    | Yes           | Sometimes          | Rarely        |

# When to Use?

## ArrayList

Use when:

* Frequent reading
* Random access
* LeetCode problems
* Spring Boot applications

Example:

```text
Student List
Product List
Employee Records
```

---

## LinkedList

Use when:

* Frequent insertions
* Frequent deletions
* Queue
* Deque
* Browser History

Example:

```text
Undo/Redo
Browser Navigation
Playlist Navigation
```

---

## Vector

Use when:

* Multiple threads access the same list
* Legacy Java code

Example:

```text
Old Enterprise Applications
Legacy Banking Systems
```

# Interview Answer

Which is fastest?

```text
ArrayList > Vector > LinkedList
```

for random access.

Which is thread-safe?

```text
Vector
```

Which is most used in real projects?

```text
ArrayList
```

Which consumes more memory?

```text
LinkedList
```

because every node stores:

Data + Previous Pointer + Next Pointer

**Next important collection:** `HashMap` (most asked in interviews), then `HashSet`, `TreeSet`, and `PriorityQueue`.
